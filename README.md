# 5A AI - نظام تحليل تسديدات البلياردو الذكي

## نظرة عامة
نظام متقدم يستخدم الذكاء الاصطناعي لتحليل وتحسين تسديدات البلياردو باستخدام نظام القياسات الاحترافي 5A.

## الميزات الرئيسية

### 1. **نظام القياسات المتقدم (5A)**
- دعم أنظمة مختلفة: 1، 2، 3، و 4 جدران
- قياسات دقيقة للكرة البيضاء (1.25 - 8)
- قياسات الهدف (1 - 12)
- قياسات القوة والمسار

### 2. **محرك الذكاء الاصطناعي**
- تحليل ذكي للضربات
- تنبؤات نسبة النجاح
- توصيات مخصصة
- تعلم من الضربات السابقة

### 3. **مكتبة ضربات ذكية**
- حفظ وتنظيم الضربات
- البحث والتصفية
- إحصائيات تفصيلية
- تتبع التطور

### 4. **تدريبات متقدمة**
- تمارين مخصصة حسب المستوى
- تدريبات تفاعلية
- قياس التقدم

## البنية التقنية

### الملفات الأساسية

| الملف | الوصف |
|------|--------|
| `index.html` | الواجهة الرئيسية |
| `5a-core.js` | نظام التحكم الأساسي |
| `5A-ai.js` | محرك الذكاء الاصطناعي |
| `5A-style.css` | الأنماط المتقدمة |
| `table-manager.js` | مدير النظام الجديد |
| `advanced-tables.js` | جداول البلياردو المتقدمة |
| `rail-systems.js` | أنظمة الجدران |
| `geometry-calculator.js` | حسابات الهندسة |
| `validation-engine.js` | محرك التحقق من الصحة |

### العمليات الخلفية (Web Workers)

- **ai-worker.js**: معالجة حسابات الذكاء الاصطناعي
- **geometry-worker.js**: حسابات الهندسة المعقدة

### إدارة البيانات

- **cloud-sync-manager.js**: مزامنة البيانات مع السحابة
- **backup-manager.js**: النسخ الاحتياطي التلقائي
- **table-state-manager.js**: إدارة حالة الجداول

### التحسينات

- **performance-optimizer.js**: تحسين الأداء
- **service-worker.js**: خدمة العاملين للتطبيقات المتقدمة

## التشغيل

### المتطلبات
- متصفح حديث يدعم:
  - ES6+
  - localStorage
  - Web Workers
  - Service Workers

### الخطوات

1. **تحميل الملفات**
```bash
git clone <repository>
cd 5A-ai
```

2. **تشغيل محلي**
```bash
python -m http.server 8000
```

3. **فتح التطبيق**
```
http://localhost:8000
```

2. أدخل قياسات الكرة البيضاء والهدف
3. أدخل قوة الكيو والمسار
4. اضغط "تحليل" للحصول على النتائج

- عرض الإحصائيات التفصيلية

### التدريبات
- اختر مستوى التدريب
- اتبع التعليمات
- اطلب توصيات ذكية

## البيانات والتخزين

- **localStorage**: تخزين محلي للبيانات
- **IndexedDB**: قاعدة بيانات محلية متقدمة
- **Cloud Sync**: مزامنة اختيارية مع السحابة

## الأداء

### تحسينات الأداء
- **Virtual Scrolling**: للجداول الكبيرة
- **Web Workers**: للحسابات الثقيلة
- **Debouncing/Throttling**: لأحداث الإدخال
- **Caching متعدد المستويات**: Memory → IndexedDB → localStorage

### معايير الأداء
- وقت التحميل: < 2 ثانية
- استجابة الواجهة: < 100ms
- استهلاك الذاكرة: < 100MB
